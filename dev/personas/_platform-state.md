# CivicForge Platform State — February 2026

Current state of the platform as experienced by test users. This is what exists right now.

- **Wicks Landing** — Carmel, IN (ZIP 46032)
- Founded by Victor (dev account)

## Core Features (Live)

### Needs Board
- Post needs or offers with categories: Home Repair, Yard & Garden, Childcare, Pet Care, Transportation, Tech Help, Cooking & Meals, Tutoring, Moving Help, Errands, Companionship, Other
- AI-assisted post extraction from natural language
- AI matching suggests responders based on skill/availability overlap
- Responses (accept/decline workflow)
- Thanks system (from post author to responder)
- Post flagging (any member can flag; 3 flags auto-hides)
- Photo uploads (EXIF stripped, max 4 per post)

### Quests (Ascendant V2.5)
- Five difficulty tiers: Spark (5 XP, self-report), Ember (15 XP, peer confirm), Flame (35 XP, photo + peer), Blaze (75 XP, community vote 3+), Inferno (150 XP, vote + evidence)
- Quest creation from natural language (AI-extracted)
- Quest claiming and completion workflow
- Peer validation for Ember+ quests
- XP awards on validation

### Skill Domains (The Forge)
- Seven domains: Craft, Green, Care, Bridge, Signal, Hearth, Weave
- XP tracked per domain via quest completion
- Logarithmic progression (fast early levels, mastery takes effort)
- Skills private by default

### Renown Tiers
- Tier 1 Newcomer: Browse, post needs, respond
- Tier 2 Neighbor: Post offers, create quests, join parties (requires invitation + 1 quest)
- Tier 3 Pillar: Create guilds, moderate (requires 50+ renown, 2 Pillar vouches)
- Tier 4 Keeper: Governance, propose rules (requires 200+ renown, 6 months, steward experience)
- Tier 5 Founder: Cross-community coordination (requires 500+ renown, cross-domain, mentoring)

### Privacy Tiers
- Ghost: Badge only
- Quiet: Tier + domain summary (DEFAULT)
- Open: Full profile visible
- Mentor: Full profile + availability

### Endorsements & Vouching
- Peer endorsements (domain-specific)
- Vouching for tier promotion (Pillars can vouch)
- Auto-promotion when vouch threshold met

### Guilds
- Created by Pillars (T3+)
- Charter with mission, domain focus, sunset clause (default 1 year)
- Steward elections (6-month terms, max 2 consecutive)
- Member management

### Governance
- Proposals with deliberation and voting phases
- Quadratic voting for charter amendments
- Categories: charter amendment, quest template, threshold change, seasonal quest, rule change, guild charter, game design, federation, other
- Voting open for configurable duration

### AI Features
- Post extraction (natural language to structured post)
- Profile matching (skill/availability-based)
- Quest extraction (natural language to quest)
- Quest matching (skills to open quests)
- Personal AI advocate (chat-based community guide)
- Content moderation (automated flag review)
- Governance analysis (proposal impact analysis)
- Rate limited: 10 req/min/user, daily token budget

### The Commons (Public Dashboard)
- Public visualization at /commons (no auth required)
- Knowledge graph, domain radar, renown pyramid, quest activity, guild health, governance gauge, community growth
- K-anonymity threshold = 3 (aggregates with < 3 suppressed)
- ISR with 5-minute revalidation

### Auth & Security
- Magic link + Google OAuth login
- Onboarding flow (community selection + profile setup)
- CSP headers, EXIF stripping, rate limiting
- GPC (Global Privacy Control) detection honored

## Known Limitations / Rough Edges
- Small community (4 test members) makes matching sparse
- No direct messaging between users
- No notification system (email or push) for responses/mentions
- Quest validation requires manual peer confirmation (no automated checks)
- Guild discovery is basic (no search, no browse-all)
- Governance proposals have no discussion/comment thread
- No mobile-optimized views (responsive but not native)
- The Commons suppresses most data at < 10 members
- No way to "undo" a flag — once flagged, it's flagged
- Activity feed is community-scoped but can feel noisy
