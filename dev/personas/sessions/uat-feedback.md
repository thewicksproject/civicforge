# Session Template: UAT Scored Feedback

## Format

The developer will describe a specific flow or feature for you to evaluate. Score each dimension 1-5 with a brief narrative explaining the score.

## Scoring Guide

| Score | Meaning |
|-------|---------|
| 1 | Broken / hostile / would make me stop using this |
| 2 | Frustrating / confusing / I'd complain to someone |
| 3 | Fine / it works but nothing special / some friction |
| 4 | Good / smooth / I'd feel positive about this |
| 5 | Excellent / delightful / I'd tell someone about this |

## Response Structure

### Usability (1-5)
How easy was this to understand and use? Could you figure it out without help? Score and 2-3 sentences.

### Usefulness (1-5)
Does this actually help you do something you care about? Does it solve a real problem in your life? Score and 2-3 sentences.

### Emotional Response (1-5)
How did this make you feel? Did you feel respected, empowered, confused, surveilled, delighted? Score and 2-3 sentences.

### Trust (1-5)
Do you trust this feature with your information / actions / community standing? Does it feel safe? Score and 2-3 sentences.

### Specific Issues
List any concrete problems you encountered or anticipate. For each:
- **What:** Describe the issue
- **Severity:** Blocker (can't proceed) / Major (painful but workable) / Minor (annoying) / Cosmetic (nitpick)
- **Context:** When/where this happens

### Suggested Changes
1-3 specific changes that would improve this for you. Be concrete — "make the button bigger" not "improve the UX."

### Would You Recommend This?
If a neighbor asked "Should I try CivicForge?" and this was the first thing they'd see, would it help or hurt your recommendation? One sentence.
